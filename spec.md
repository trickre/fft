# FFT Web App 設計書

## 1. 概要

本アプリは、スマートフォンおよびPCブラウザで動作する、クライアントサイド完結型のFFT表示アプリである。  
端末のマイクから取得した音声入力をリアルタイムにFFT解析し、その結果を画面上に常時表示する。  
FFT結果の保存、ユーザー管理、サーバー通信は行わないシンプルな構成とする。

## 2. 目的

- ブラウザ上で手軽に音の周波数成分を確認できるようにする
- スマートフォンとPCの両方で利用できるようにする
- 学習用途、簡易計測、デバッグ用途に使える最小構成のFFT表示アプリを提供する
- 実装と保守を簡潔に保つ

## 3. スコープ

### 3.1 対象

- マイク入力の取得
- リアルタイムFFT解析
- 周波数スペクトルの可視化
- スマホ/PCブラウザ向けレスポンシブ表示
- FFT表示に関する最小限のパラメータ変更

### 3.2 対象外

- FFT結果の保存
- ログイン/認証
- サーバーサイドAPI
- DB保存
- ユーザーごとの設定同期
- 録音データのアップロード

## 4. 想定利用環境

### 4.1 対応デバイス

- iPhone / Android
- PC / Mac

### 4.2 対応ブラウザ

- iOS Safari 最新系
- Android Chrome 最新系
- Chrome / Edge / Safari / Firefox の主要最新版

### 4.3 利用条件

- マイク利用を許可できること
- HTTPS 環境またはローカル開発環境で動作すること

補足:
ブラウザの `getUserMedia` は、通常セキュアコンテキストでのみ安定して利用できる。

## 5. 要件

### 5.1 機能要件

- 起動後、ユーザーがマイク利用を許可できること
- マイク入力をリアルタイムに取得できること
- 入力音声に対してFFTを継続的に実行できること
- FFT結果を画面上に常時更新表示できること
- スマホでもPCでも視認できるUIであること
- 解析停止/再開ができること
- 最低限の解析条件をUIから変更できること

### 5.2 最低限の操作項目

- マイク開始
- 停止
- FFTサイズ変更
- 表示レンジ切替
- 入力レベルまたはピーク周波数の簡易表示

### 5.3 非機能要件

- 画面操作に対して軽快に反応すること
- 低遅延で表示更新されること
- 複雑な初期設定なしで利用開始できること
- クライアントサイドのみで完結すること

## 6. 技術構成

### 6.1 採用方針

過剰な構成を避け、以下を推奨する。

- フロントエンド: `React` + `TypeScript` + `Vite`
- スタイリング: `CSS Modules` またはシンプルな `CSS`
- 音声処理: `Web Audio API`
- 描画: `Canvas API`
- 配置: 静的ホスティング

### 6.2 採用理由

- `Vite` はクライアントサイド完結の小規模アプリと相性がよい
- `React` は画面状態の制御を整理しやすい
- `Web Audio API` はブラウザ標準でマイク入力と周波数解析を扱える
- `Canvas API` はスペクトル表示を軽量に描画しやすい
- サーバーやDBを不要にできる

### 6.3 採用しないもの

初期版では以下を採用しない。

- `Next.js`
- サーバーサイドAPI
- DB
- 認証
- 状態管理ライブラリ
- 外部可視化ライブラリ

## 7. システム構成

### 7.1 全体構成

```text
[ Browser ]
  ├─ UI (React)
  ├─ Microphone Input (getUserMedia)
  ├─ Audio Processing (Web Audio API / AnalyserNode)
  └─ Spectrum Rendering (Canvas)
```

### 7.2 構成方針

- 単一のフロントエンドアプリとして構成する
- すべての処理をブラウザ内で完結させる
- FFTデータはメモリ上のみで扱い、保存しない

## 8. 音声処理設計

### 8.1 処理フロー

1. ユーザーがマイク開始を実行
2. `navigator.mediaDevices.getUserMedia({ audio: true })` で入力取得
3. `AudioContext` を生成
4. `MediaStreamAudioSourceNode` を作成
5. `AnalyserNode` に接続
6. 周期的に周波数データを取得
7. `Canvas` にスペクトルを描画

### 8.2 FFT実装方針

初期版では `Web Audio API` の `AnalyserNode` を利用する。

理由:

- ブラウザ標準で利用可能
- 実装コストが低い
- 本用途では十分な精度と速度を期待できる

将来的に独自FFT処理が必要になった場合のみ、別実装を検討する。

### 8.3 主な調整項目

- `fftSize`
- `minDecibels`
- `maxDecibels`
- `smoothingTimeConstant`

ただし初期UIでは、ユーザーが混乱しやすい項目を増やしすぎない。  
初期公開では `fftSize` と表示レンジ程度に留める。

## 9. UI設計

### 9.1 画面構成

単一画面構成を基本とする。

- ヘッダー
- スペクトル表示エリア
- 現在値表示エリア
- 操作パネル
- エラーメッセージ表示エリア

### 9.2 表示内容

- 横軸: 周波数
- 縦軸: 振幅またはレベル
- スペクトルバーまたはライン
- ピーク周波数
- 入力状態
- マイク許可状態

### 9.3 操作UI

- `開始`
- `停止`
- `FFTサイズ`
- `表示モード` または `周波数レンジ`

### 9.4 レスポンシブ方針

- スマホでは縦積みレイアウト
- PCではスペクトル表示領域を広く取る
- ボタンはタップしやすいサイズを確保する
- 数値やラベルは屋外利用も考慮し視認性を優先する

## 10. 状態管理

### 10.1 管理対象

- マイク利用状態
- 解析中/停止中
- FFT設定値
- 現在の周波数データ
- エラー状態

### 10.2 実装方針

- `React` の `useState` と `useRef` を基本とする
- 音声ノードや `AudioContext` は `ref` で保持する
- グローバル状態管理ライブラリは使わない

## 11. 主要コンポーネント案

```text
App
├─ Header
├─ SpectrumCanvas
├─ StatusPanel
├─ ControlPanel
└─ ErrorNotice
```

### 11.1 役割

- `App`: 全体状態管理
- `SpectrumCanvas`: FFT結果の描画
- `StatusPanel`: ピーク周波数や入力状態表示
- `ControlPanel`: 開始/停止やFFTサイズ変更
- `ErrorNotice`: 権限拒否や非対応時の表示

## 12. パフォーマンス方針

- 描画は `requestAnimationFrame` を基本とする
- 再描画コストを抑えるため、FFT描画は `Canvas` を利用する
- Reactの状態更新を毎フレーム大量に行いすぎない
- 周波数配列の取得と描画を分離し、必要最小限のUI更新に留める

## 13. エラーハンドリング

### 13.1 想定エラー

- マイク権限が拒否された
- ブラウザがマイクAPIに非対応
- `AudioContext` の開始に失敗
- モバイル端末で自動開始制約に抵触

### 13.2 対応方針

- エラー内容をUIで簡潔に表示する
- 再試行できる導線を用意する
- ユーザー操作後に音声処理を開始する

## 14. セキュリティ・プライバシー

- 音声データは外部送信しない
- FFT結果は保存しない
- ログにも音声内容を残さない
- 利用開始前にマイク使用目的を明示する

## 15. 開発方針

### 15.1 ディレクトリ構成案

```text
/
├─ src/
│  ├─ components/
│  ├─ hooks/
│  ├─ lib/
│  ├─ styles/
│  ├─ App.tsx
│  └─ main.tsx
├─ public/
└─ index.html
```

### 15.2 実装分離方針

- `hooks/`: マイク入力やFFT取得処理
- `components/`: UI部品
- `lib/`: 周波数軸計算や表示補助関数
- `styles/`: 全体スタイル

### 15.3 テスト方針

- ロジック部分は単体テスト対象とする
- マイク連携の一部はブラウザ依存のため手動確認を前提とする
- 少なくとも以下は確認対象とする

- マイク開始/停止
- 権限拒否時表示
- FFTサイズ変更時の表示更新
- スマホ表示とPC表示

## 16. リリース方針

### 16.1 初期版で実装するもの

- 単一画面
- マイク開始/停止
- リアルタイムスペクトル表示
- FFTサイズ切替
- ピーク周波数表示
- 権限エラー表示
- レスポンシブ対応

### 16.2 初期版で見送るもの

- データ保存
- 解析履歴
- 録音機能
- 画像保存
- 高度な分析機能
- 複数画面構成

## 17. 制約事項

- HTTPS 環境での公開が前提
- 端末やブラウザによりマイク特性が異なる
- 周波数表示の精度は端末マイク性能に依存する
- バックグラウンド動作や画面スリープ時の挙動は端末依存となる

## 18. 結論

本アプリは、マイク入力をFFTしてリアルタイム表示する、クライアントサイド完結のシンプルなWebアプリとして設計する。  
構成は `React + TypeScript + Vite + Web Audio API + Canvas` を基本とし、保存・認証・サーバーを持たない最小構成とする。  
まずは単一画面で安定して動作することを優先し、必要が明確になった段階で表示機能や分析補助機能を追加する。
